# 직원 수정 폼 입사일(hireDate) 표시 오류 수정

## 문제 / 동기
직원 수정 폼(`/employees/{id}/edit`)에서 `입사일` 필드가 항상 빈 값으로 보인다.
원인은 `EmployeeForm.hireDate`(`LocalDate`)에 날짜 포맷 지정이 없어, Thymeleaf가
`th:field`로 값을 렌더링할 때 기본 로케일 포맷(예: `26. 1. 1.`)을 사용하기 때문이다.
`<input type="date">`는 값이 반드시 `yyyy-MM-dd`(ISO) 형식이어야 인식하므로, 로케일
포맷 문자열은 브라우저가 무시해 빈 칸으로 보인다.

## 요구사항
- `EmployeeForm.hireDate`에 ISO(`yyyy-MM-dd`) 포맷을 명시적으로 지정해, 수정 폼에서
  기존 입사일 값이 `<input type="date">`에 정상적으로 채워지도록 한다.
- 등록 폼(신규 등록, 초기값 없음)과 수정 폼(기존 값 있음) 모두에서 날짜 입력·제출이
  이전과 동일하게 정상 동작해야 한다(파싱/검증 회귀 없음).
- 다른 필드, 검증 규칙, 화면 레이아웃은 변경하지 않는다.

## 수용 기준
- [ ] 수정 폼 진입 시 `입사일` 입력창에 기존 값이 `yyyy-MM-dd` 형식으로 채워져 보인다.
- [ ] 등록 폼에서 입사일을 선택해 제출하면 정상 저장된다.
- [ ] 수정 폼에서 입사일을 변경해 제출하면 정상 반영된다.
- [ ] 입사일 누락 시 기존과 동일한 필수값 검증 에러가 표시된다.
- [ ] 다른 필드·화면에 회귀가 없다.

## 범위 외 항목
- 날짜 형식 외 다른 필드/검증 변경.
- 폼 레이아웃 추가 변경.

## 열린 질문
없음 — 원인과 수정 방향(ISO 날짜 포맷 지정)이 명확하다.
