# 직원 목록 부서별 그룹핑

## 문제 / 동기
현재 직원 목록은 정렬 기준 없이(대체로 등록 순서) 표시되어 같은 부서 직원이 흩어져
보인다. 부서별로 묶어서 보여주면 조직 구조를 한눈에 파악하기 쉬워진다.

## 요구사항
- 직원 목록을 부서 기준으로 정렬한다(부서 내에서는 이름 기준 보조 정렬).
- 기존 하나의 표(카드로 감싼 `list.html`)는 그대로 유지하고, 표 안에서 부서가 바뀌는
  지점마다 부서명을 표시하는 구분 행(그룹 헤더 행)을 넣는다.
- 그룹 헤더 행은 나머지 데이터 행과 시각적으로 구분되는 스타일(굵은 글씨, 옅은 배경 등)을
  갖는다.
- 기존 zebra(줄무늬) 배경은 그룹 헤더 행 때문에 어긋나지 않도록, 실제 직원 데이터 행만
  기준으로 교대로 적용되게 조정한다.
- 정렬은 서버(컨트롤러/서비스/리포지토리) 단에서 이루어진다 — 뷰에서 임의로 재정렬하지
  않는다.
- 별도의 부서 필터/검색 UI는 추가하지 않는다(F-05 검색·필터는 여전히 범위 밖).
- 이모지는 사용하지 않는다.

## 수용 기준
- [ ] 목록이 부서 기준으로 정렬되어(부서 내 이름순) 표시된다.
- [ ] 부서가 바뀌는 지점마다 부서명을 보여주는 구분 행이 나타난다.
- [ ] 구분 행이 일반 데이터 행과 시각적으로 구분된다.
- [ ] zebra 줄무늬가 구분 행과 무관하게 실제 데이터 행 기준으로 올바르게 교대로 적용된다.
- [ ] 각 행의 상세 링크, 상태 배지 등 기존 목록 기능이 그대로 동작한다.
- [ ] 등록·수정 후 목록으로 돌아왔을 때도 동일한 정렬·그룹핑이 유지된다.

## 범위 외 항목
- 부서 필터/검색, 부서별 접기·펼치기(collapsible) 기능.
- 부서별 인원 수 집계 표시.
- 목록을 부서별 별도 카드(표)로 분리하는 방식(이번엔 하나의 표 안에서 그룹핑하기로 확정).
- 정렬 기준을 사용자가 선택하게 하는 기능(부서순 고정).

## 열린 질문
없음 — 그룹핑 방식(하나의 표 안에서 그룹핑)은 사용자와 확정했다.
