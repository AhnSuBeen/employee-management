# 직원 상세 페이지 카드형 레이아웃 개선

## 문제 / 동기
현재 상세 페이지(`templates/employee/detail.html`)는 `dl/dt/dd`로 라벨과 값이 수직으로
단순 나열되어 있어 정보 밀도가 낮고 시각적으로 밋밋하다. 목록/폼 화면은 별도 진행 중인
작업(사용자가 직접 이모지 라벨을 정리하는 중)이 있으므로 건드리지 않고, 상세 페이지만
카드/그리드 레이아웃으로 재구성해 가독성을 높인다.

## 요구사항
- `detail.html`의 `dl` 목록을 카드 컨테이너 안의 라벨-값 그리드로 재구성한다
  (부서/직급/이메일/전화번호/입사일/상태 6개 필드, 순서와 내용은 그대로 유지).
- 이모지는 사용하지 않는다 — 순수 CSS(간격, 구분선, 타이포그래피, 카드 그림자 등)로만
  개선한다. 기존에 남아있는 `✏️ 수정` 버튼의 이모지도 일관성을 위해 제거한다.
- `static/css/style.css`에 카드/그리드 관련 스타일을 추가한다. 기존 배지
  (`badge-active`/`badge-on-leave`/`badge-resigned`)와 버튼(`.btn`) 스타일은 그대로 재사용한다.
- 상태 변경 버튼 영역(`status-actions`)의 기능과 마크업 구조(폼 하나당 상태 하나)는
  그대로 유지한다 — 시각적 배치만 카드 안에 자연스럽게 어울리도록 다듬는다.
- 외부 CDN·아이콘 폰트·JS 라이브러리는 도입하지 않는다(`thymeleaf-ui` 규칙 유지).
- 기능 변경 없음 — 표시되는 데이터, 링크, 폼 동작은 이전과 동일해야 한다.

## 수용 기준
- [ ] 상세 페이지가 카드 형태의 컨테이너 안에 라벨-값 그리드로 표시된다.
- [ ] 이모지가 전혀 사용되지 않는다(기존 `✏️ 수정` 포함).
- [ ] 상태 배지, 수정 링크, 상태 변경 버튼이 기존과 동일하게 동작한다.
- [ ] 목록(`list.html`)·폼(`form.html`)·레이아웃(`layout.html`)은 변경하지 않는다.
- [ ] 브라우저에서 상세 페이지가 깨짐 없이 렌더링된다.

## 범위 외 항목
- 목록 페이지, 등록/수정 폼 페이지, 공통 레이아웃(nav) 변경 — 사용자가 별도로 진행 중.
- 이모지 아이콘 도입(기존 `employee-ui-polish` 스펙의 이모지 방향과는 별개로, 이번엔
  이모지 없이 진행하기로 확정).
- 다크모드, 반응형 레이아웃 등 추가 UI 기능.
- 신규 필드나 기능 추가.

## 열린 질문
없음 — 카드/그리드 방향, 이모지 미사용 모두 사용자와 확정했다.
