# 직원 등록/수정 폼 2단 그리드 레이아웃

## 문제 / 동기
직원 등록/수정 폼(`form.html`)은 라벨-입력칸이 세로로 하나씩 쌓여 있어 필드 수(6개)에
비해 페이지가 세로로 길다. 상세 페이지의 카드/그리드 개선과 통일감을 갖도록 폼도
2단 그리드로 재배치하고, 입력칸 자체의 스타일(둥근 모서리, 포커스 강조)도 함께
다듬는다.

## 요구사항
- 폼 필드(이름/부서/직급/이메일/전화번호/입사일) 6개를 2열 그리드로 배치한다. 필드
  순서·`name`/`th:field` 바인딩·검증 로직은 변경하지 않는다.
- 저장 버튼은 그리드 아래 전체 너비로 배치한다.
- 입력창(`input`) 스타일을 개선한다: 둥근 모서리, 테두리 색, 포커스 시 강조(테두리
  색 변경 + 옅은 그림자).
- 에러 메시지는 각 필드 아래 그대로 유지한다(위치·문구 변경 없음).
- 등록(`/employees/new`)·수정(`/employees/{id}/edit`) 두 화면 모두 동일하게 적용한다
  (하나의 템플릿 `form.html`을 공유하므로 자동으로 함께 적용됨).
- 목록/상세 페이지는 이번 변경 대상이 아니다.
- 이모지는 사용하지 않는다. 외부 CDN·아이콘 폰트·JS 라이브러리는 도입하지 않는다.

## 수용 기준
- [ ] 등록 폼이 2단 그리드로 표시된다.
- [ ] 수정 폼도 동일한 2단 그리드로 표시된다.
- [ ] 입력창에 둥근 모서리와 포커스 강조 스타일이 적용된다.
- [ ] 필수값 누락·형식 오류 시 에러 메시지가 이전과 동일하게 필드 아래 표시된다.
- [ ] 등록/수정 제출이 이전과 동일하게 동작한다(정상 저장, 검증 실패 시 같은 화면
      복귀).
- [ ] 목록/상세 페이지에 영향이 없다.

## 범위 외 항목
- 필드 추가/삭제, 검증 규칙 변경.
- 목록·상세 페이지 추가 변경.
- 반응형(모바일) 레이아웃.

## 열린 질문
없음 — 2단 그리드 + 입력칸 스타일 개선 방향을 사용자와 확정했다.
