# 목록 · 폼 페이지 카드 레이아웃 통일

## 문제 / 동기
상세 페이지(`detail.html`)는 카드 컨테이너(`.detail-card`) 안에 내용이 표시되도록 개선됐지만,
목록(`list.html`)과 등록/수정 폼(`form.html`)은 여전히 배경색 없는 페이지 위에 표/폼이 바로
놓여 있어 시각적으로 통일되어 있지 않다. 세 화면 모두 동일한 카드 컨테이너 스타일을
사용하도록 통일한다.

## 요구사항
- 목록 페이지: 기존 `<table>` 구조는 그대로 두고, 카드 컨테이너로 감싼다(표 자체를 개별
  카드형 그리드로 재구성하지 않는다 — 정렬·링크 등 기존 표 동작 유지).
- 폼 페이지(등록/수정 공용): 기존 `<form>` 구조는 그대로 두고, 카드 컨테이너로 감싼다.
- 상세 페이지에서 사용 중인 카드 컨테이너 스타일(`.detail-card`)을 세 화면에서 공용으로
  쓸 수 있도록 이름을 일반화한다(예: `.card`). 상세 페이지의 라벨-값 그리드
  (`.detail-grid`/`.detail-label`/`.detail-value`)는 상세 페이지 전용으로 그대로 둔다.
- 표가 카드 안에 들어가면서 표 자체의 그림자·배경이 카드와 중복되지 않도록 조정한다.
- 이모지는 사용하지 않는다(기존 방향 유지).
- 외부 CDN·아이콘 폰트·JS 라이브러리는 도입하지 않는다.
- 기능 변경 없음 — 목록 정렬/링크, 폼 검증·제출 동작은 이전과 동일해야 한다.

## 수용 기준
- [ ] 목록 페이지의 표가 카드 컨테이너 안에 표시된다(표 기능은 동일).
- [ ] 폼 페이지(등록/수정)의 입력 필드가 카드 컨테이너 안에 표시된다.
- [ ] 세 화면(목록/상세/폼)이 동일한 카드 컨테이너 스타일(배경·그림자·모서리)을 공유한다.
- [ ] 이모지가 사용되지 않는다.
- [ ] 목록의 상세 링크, 상태 배지, 폼 검증 오류 표시, 저장/등록 흐름이 이전과 동일하게
      동작한다.

## 범위 외 항목
- 목록을 개별 카드형 그리드로 재구성(표 유지, 컨테이너만 카드화).
- 페이지네이션, 검색·필터 등 신규 기능.
- 상세 페이지 자체의 추가 변경(이미 완료됨).

## 열린 질문
없음 — 목록은 "표를 카드로 감싸기" 방식으로 사용자와 확정했다. 폼도 동일하게 카드로
감싸는 것으로 가정한다(상세 페이지와의 통일이 목적이므로 별도 이견이 없는 한 동일 적용).
