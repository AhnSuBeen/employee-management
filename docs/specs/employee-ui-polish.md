# 직원관리 UI 개선 (아이콘 · 색상 톤)

## Problem / motivation
F-01~F-04 기능은 완성됐지만 화면이 단조롭다(흑백 표, 텍스트만 있는 배지). 아이콘과 색상
톤을 추가해 가독성과 완성도를 높인다. 순수 시각적 개선이며 기능 변경은 없다.

## Requirements
- 상태 배지에 유니코드 이모지 추가: 재직(`ACTIVE`) 🟢, 휴직(`ON_LEAVE`) 🟡, 퇴직(`RESIGNED`) ⚪
  — 이모지와 텍스트 라벨을 함께 표시한다(이모지 단독 표시는 금지, 접근성 고려).
- 목록/상세/폼 화면의 필드 라벨 옆에 유니코드 이모지 아이콘을 추가한다
  (예: 👤 이름, 🏢 부서, 💼 직급, ✉️ 이메일, 📞 전화번호, 📅 입사일).
- 색상 톤을 차분하게 개선한다: 헤더/네비게이션 바는 짙은 네이비 배경, 버튼은 파란 계열
  강조색, 목록 표에는 zebra(줄무늬) 배경.
- 외부 CDN·아이콘 폰트·JS 라이브러리를 도입하지 않는다(`thymeleaf-ui` 규칙 유지) — CSS와
  유니코드 문자만 사용한다.
- 기존 기능(F-01~F-04) 동작에는 변화가 없다 — 순수 시각적 개선만 다룬다.

## Acceptance criteria
- [ ] 목록 화면 상태 배지에 이모지 + 텍스트가 함께 표시된다
- [ ] 헤더/네비게이션에 네이비 색상이 적용된다
- [ ] 목록 표에 zebra 스타일이 적용된다
- [ ] 상세/폼 화면 필드 라벨에 이모지 아이콘이 적용된다
- [ ] 버튼(등록/저장/상태 변경)에 파란 계열 강조 스타일이 적용된다
- [ ] 기존 F-01~F-04 동작에 회귀가 없다(기존 HTTP 검증 시나리오 재확인)

## Out of scope
- 다크모드, 반응형 레이아웃 등 추가 UI 기능
- 아이콘 폰트/이미지 자산 도입
- 신규 화면·필드 추가

## Open questions
없음 — 아이콘 방식(유니코드/이모지)과 색상 톤(차분한 톤: 네이비 헤더 + 파란 버튼 + zebra
표) 모두 사용자와 확정했다.
