# 직원 목록 검색/필터 (이름 · 부서 · 직급)

## 문제 / 동기
원래 요구사항 정의서(`docs/specs/employee-management-spec.md`)에서는 F-05(검색·필터)를
이번 구현 범위에서 제외하기로 확정했었다. 이제 사용자가 목록 페이지에 이름/부서/직급
기준 검색 기능을 명시적으로 요청했으므로, F-05를 이번 스펙으로 다시 범위에 포함해
진행한다. 목록 화면 왼쪽에 검색 사이드바를 추가해 조건에 맞는 직원만 필터링해 보여준다.

## 요구사항
- 목록 페이지(`/employees`) 왼쪽에 검색 사이드바(폼)를 추가한다. 입력 필드: 이름, 부서,
  직급 (3개, 모두 선택 입력— 비워두면 해당 조건은 적용하지 않는다).
- 검색은 서버 사이드에서 처리한다: 사이드바 폼을 GET으로 제출하면 `/employees`가 쿼리
  파라미터(`name`, `department`, `position`)를 받아 필터링된 결과로 페이지를 다시
  렌더링한다. 클라이언트 JS 필터링은 사용하지 않는다.
- 세 필드를 동시에 입력하면 AND로 결합한다(모두 만족하는 직원만 표시).
- 각 필드는 부분 일치(포함) + 대소문자 무관으로 검색한다(예: 부서에 "개발" 입력 시
  "개발팀" 포함 직원 표시).
- 검색 결과에도 기존 부서별 그룹핑·정렬(부서→이름순)과 zebra 스타일이 동일하게 적용된다.
- 사이드바에는 검색 버튼과 초기화(전체 목록으로 돌아가기) 링크를 둔다.
- 검색 후에도 사이드바 입력값이 유지되어야 한다(방금 검색한 조건이 그대로 채워져 있음).
- 이모지는 사용하지 않는다. 외부 CDN·아이콘 폰트·JS 라이브러리는 도입하지 않는다.
- 목록 외 화면(상세/폼)에는 영향이 없다.

## 수용 기준
- [ ] 목록 페이지 왼쪽에 이름/부서/직급 입력 필드가 있는 검색 사이드바가 보인다.
- [ ] 하나의 필드만 입력해도 해당 조건으로 부분 일치 검색이 동작한다.
- [ ] 두 개 이상의 필드를 입력하면 AND로 결합되어 모두 만족하는 직원만 보인다.
- [ ] 검색 결과에도 부서별 그룹 헤더와 zebra 스타일이 정상 적용된다.
- [ ] 검색 결과가 없을 때 빈 상태를 안내하는 문구가 표시된다.
- [ ] 검색 후 사이드바에 방금 입력한 값이 그대로 남아 있다.
- [ ] "초기화" 클릭 시 전체 목록으로 돌아간다.
- [ ] 기존 상세 링크, 상태 배지, 등록/수정 흐름에 회귀가 없다.

## 범위 외 항목
- 상태(재직/휴직/퇴직) 조건 검색 — 이번엔 이름/부서/직급 3개만 다룬다.
- 자동완성, 실시간(입력 중) 검색, 정렬 기준 변경 UI.
- 페이지네이션.
- 검색 조건 저장(즐겨찾기 등).

## 열린 질문
없음 — 서버사이드 폼 제출 방식, AND 결합, 부분/대소문자 무관 일치 모두 사용자와
확정했다(부분 일치·대소문자 무관은 일반적인 검색 UX로 가정 — 이견 있으면 알려달라).
